## 📖 Overview

**TestPilot AI** is an intelligent test automation platform designed to verify applications generated by AI programming tools (Cursor, Windsurf, etc.). It combines **blueprint-driven testing** with **multimodal AI visual analysis** to test like a human tester.

### 🎯 Key Features

- 🎯 **Blueprint-Driven Testing** - AI programming tools generate test blueprints (`testpilot.json`), TestPilot executes them with pixel-perfect precision
- 👁️ **Visual AI Analysis** - Multimodal AI analyzes screenshots to detect UI bugs, layout issues, and unexpected behaviors
- 🔧 **Auto-Repair Loop** - Automatically fixes bugs, re-tests, and iterates until 100% pass rate (fix-test-fix cycle)
- 🌐 **Multi-Platform Support** - Web, Android, iOS, Windows Desktop, macOS, WeChat Mini Programs
- 🧠 **Memory System** - Learns from test history, accumulates testing experience in local SQLite database
- 🔌 **IDE Integration** - VSCode/Windsurf extension for seamless workflow
- 📊 **Real-time Monitoring** - WebSocket live logs, VNC/screenshot streaming, step-by-step progress tracking

## 🚀 Quick Start

### Prerequisites

- **Python 3.10+**
- **Poetry** (dependency management)
- **Docker Desktop** (for web testing sandbox, optional for mobile/desktop)

### Installation

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/TestPilotAI.git
cd TestPilotAI

# Install dependencies (auto-creates virtual environment)
poetry install

# Install Playwright browsers (for web testing)
poetry run playwright install chromium

# Configure environment variables
cp .env.example .env
# Edit .env and add your AI API key
```

### Configure AI Model

TestPilot AI uses multimodal AI for visual analysis. Supported providers:

- **Doubao (豆包)** - Recommended, optimized for Chinese UI (default)
- **OpenAI GPT-4 Vision** - Compatible via OpenAI SDK
- **Other OpenAI-compatible APIs** - Any provider supporting vision models

Edit `.env`:

```bash
TP_AI_API_KEY=your_api_key_here
TP_AI_API_BASE_URL=https://ark.cn-beijing.volces.com/api/v3  # or OpenAI endpoint
TP_AI_MODEL=doubao-seed-1-8-251228  # or gpt-4-vision-preview
```

### Start the Engine

```bash
# Start the core engine
poetry run python main.py

# Or use hot-reload mode (for development)
poetry run uvicorn src.app:app --reload --port 8900
```

Visit **http://127.0.0.1:8900/docs** for interactive API documentation.

## 📚 How It Works

### 1️⃣ Blueprint Generation

AI programming tools (Cursor/Windsurf) automatically generate test blueprints while coding:

```json
{
  "app_name": "Shop Demo",
  "platform": "web",
  "base_url": "http://localhost:8080",
  "pages": [
    {
      "url": "/",
      "name": "Home Page",
      "scenarios": [
        {
          "name": "User Login Success",
          "steps": [
            {"action": "navigate", "value": "/", "description": "Open home page"},
            {"action": "fill", "target": "#username", "value": "testuser"},
            {"action": "fill", "target": "#password", "value": "password123"},
            {"action": "click", "target": "#loginBtn"},
            {"action": "assert_text", "expected": "Welcome", "description": "Verify login success"}
          ]
        }
      ]
    }
  ]
}
```

### 2️⃣ Test Execution

TestPilot executes the blueprint with **UI Tree + AI Visual Dual Insurance**:

- **UI Tree Priority** - Fast, pixel-accurate element location (CSS selector, XPath, accessibility ID)
- **AI Visual Fallback** - When UI tree fails, AI analyzes screenshots to find elements by visual appearance
- **Auto-Scroll Search** - Automatically scrolls to find off-screen elements

### 3️⃣ Bug Detection

Multimodal AI analyzes screenshots at each step:

- ✅ **Visual Comparison** - Expected vs Actual UI state
- ✅ **Text Recognition** - OCR for error messages, labels, buttons
- ✅ **Layout Analysis** - Detect misaligned elements, overlapping content
- ✅ **Anomaly Detection** - Crash dialogs, blank screens, loading spinners

### 4️⃣ Auto-Repair Loop (Optional)

When bugs are found, TestPilot can automatically fix them:

1. **Bug Classification** - Categorize bugs (selector error, logic bug, timing issue, etc.)
2. **AI Code Fix** - Generate code patches using AI
3. **Apply Patch** - Automatically apply fixes to source code
4. **Re-Test** - Run tests again to verify fix
5. **Iterate** - Repeat until 100% pass rate

## 📡 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/test/run` | **Run blueprint test** (supports `auto_repair` parameter) |
| POST | `/api/v1/test/mobile-blueprint` | Run mobile blueprint test (Android/iOS) |
| POST | `/api/v1/test/desktop-blueprint` | Run desktop blueprint test (Windows/macOS) |
| POST | `/api/v1/test/quick` | Quick exploratory test (no blueprint required) |
| GET | `/api/v1/memory/history` | Query test history |
| GET | `/api/v1/memory/stats` | Memory system statistics |
| POST | `/api/v1/community/share` | Share test experience to community |
| GET | `/api/v1/community/suggest` | Get similar bug fix suggestions |

Full API documentation: **http://127.0.0.1:8900/docs**

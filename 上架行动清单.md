## 一、上架平台决策

| 平台 | 是否上架 | 理由 | 网站 |
|------|---------|------|------|
| VS Code Marketplace | ✅ 首选 | 全球开发者最多、生态最开放、免费上架 | https://marketplace.visualstudio.com |
| Open VSX Registry | ✅ 强烈推荐 | 一次打包同时覆盖 Cursor / Windsurf / Gitpod | https://open-vsx.org |
| JetBrains Plugin Repository | ❌ 暂不考虑 | 需要用 Java/Kotlin 重新开发，不是上传.vsix那么简单 | https://plugins.jetbrains.com |
| GitHub Marketplace | ⚠️ 远期考虑 | 等与 GitHub CI/CD 深度集成后再评估 | https://github.com/marketplace |

**结论**：先打 VS Code + Open VSX 双平台，一份.vsix包两用，覆盖 90% 目标用户。

## 二、收款/支付方案

> 你有公司主体+对公账户 → 优先走 Paddle（全托管）

| 工具 | 类型 | 作用 | 网站 |
|------|------|------|------|
| Paddle | 全球支付处理器（Merchant of Record） | 向用户收钱、自动处理全球VAT/税务、开发票 | https://www.paddle.com |
| Airwallex（空中云汇） | 跨境收款账户 | Paddle 打来的外币 → 换汇 → 入国内对公账户 | https://www.airwallex.com/cn |

> **两者是上下游关系，搭配使用：Paddle 收钱 → Airwallex 换汇入账**
> 不要选 Lemon Squeezy（面向个人开发者，企业合规支持弱）

**Paddle 注册需要准备：**
- 营业执照（扫描件）
- 对公账户信息（开户行 + 账号）
- 公司官网（需要境外可访问）
- 产品描述 + 定价方案

**审核周期：** 通常 3-7 个工作日，准备好材料再提交。

## 六、行动清单（按顺序执行）

### 阶段一：本周内（上架前置条件）

```
□ 1. 在官网添加 Privacy Policy 页面（/privacy）
□ 2. 注册 VS Code Marketplace Publisher 账号
□ 3. 写好插件英文简介（displayName + description，我可帮润色）
□ 4. 检查 extension/package.json 的 icon、keywords、categories 是否齐全
□ 5. 解决"难安装"问题（一键脚本 or 优先推云端版，见项目规划）
```

### 阶段二：第 2 周

```
□ 6. 执行 vsce publish 上架 VS Code Marketplace
□ 7. 执行 ovsx publish 上架 Open VSX（覆盖 Cursor/Windsurf）
□ 8. 注册 Paddle + Airwallex，开始资质审核（需 3-7 天）
□ 9. 发掘金文章 + V2EX 帖子（带演示视频链接）
```

### 阶段三：第 3-4 周

```
□ 10. Paddle 审核通过 → 配置插件 License 验证 API
□ 11. 配置 api.testpilot.xinzaoai.com 二级域名（指向后端服务器）
□ 12. 开通付费订阅计划（对应项目规划里的积分制方案）
□ 13. GitHub README 双语化（顶部英文 + 下方中文）
□ 14. 录制 B 站工具演示视频
```

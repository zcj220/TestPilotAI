### 关键技术发现（2026-03-07）
**SDK导航方法全部超时，evaluate调用原生API又快又稳：**
| 方式 | 耗时 | 可用 |
|------|------|------|
| `mp.navigateTo()` SDK | 10秒超时 | ❌ |
| `mp.navigateBack()` SDK | 10秒超时 | ❌ |
| `mp.reLaunch()` SDK | 10秒超时 | ❌ |
| `evaluate(()=>wx.navigateTo())` | 64ms | ✅ |
| `evaluate(()=>wx.navigateBack())` | 25ms | ✅ |
| `evaluate(()=>wx.reLaunch())` | 41ms | ✅ |

**场景间重置方案：** `evaluate(wx.reLaunch)` 回首页 + `evaluate` 清globalData，无需重启。

### FreshMart盲测结果（v4, commit 394f504）
- **7场景全部跑通，发现5/6 Bug，耗时31.8秒（只重启1次）**
- 🐛 会员价格：vipPrice用了0.85而非0.8
- 🐛 库存限制：addToCart只检查stock<=0，不检查购物车已有数量
- 🐛 搜索排序：用String.localeCompare做数字排序（字符串排序）
- 🐛 优惠券满减：判断条件是 > 而非 >=（小计=门槛时不能用）
- 🐛 配送费：同城配送写了5元而非0元
- ✅ 售罄商品：正确拦截
- ✅ 购物车精度：toFixed(2)掩盖了浮点误差（第6个Bug未被触发）

## 备忘
- 自动化端口固定9420（`--auto-port 9420`），不是HTTP服务端口（如32815）
- 开头用 `cli close` + `cli open` + `cli auto` 三步重启确保干净首页
- `cli auto` 单独执行不一定能清掉上次残留页面，必须先close再open

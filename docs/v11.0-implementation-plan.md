### 技术决策

| 决策 | 方案 | 原因 |
|------|------|------|
| **数据库** | 先用SQLite本地测试 | 不需要立即部署云端，本地验证逻辑正确性 |
| **云端部署** | 暂不部署，Phase 2先本地跑通 | 避免过早优化，先验证产品逻辑 |
| **向量检索** | 暂不用pgvector，用简单文本相似度 | 本地测试阶段够用，云端再上pgvector |
| **AI调用** | 匹配引擎的AI语义匹配用doubao | 成本可控，仅在前两级无结果时触发 |
| **经验粒度** | 以修复attempt为粒度，不是整个Bug | 碎片化上传，检索更精准 |
| **去重策略** | 上传时提示+定期自动合并 | 用户体验+自动化结合 |

---

### 关键设计原则

1. **只上传已验证修复**：重测通过才能分享
2. **碎片化经验**：每个attempt独立成卡片
3. **铁律通过Bug报告注入**：不依赖任何IDE配置
4. **防篡改靠引擎验证**：Bug记录+重测通过+git diff交叉验证
5. **Top 3展示**：默认显示采纳率最高的3个方案
6. **失败经验也有价值**："走过的弯路"告诉别人哪些路不通
